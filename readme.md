# 🚀 రోబోషాప్ (RoboShop) ఈ-కామర్స్ ఆటోమేషన్ రన్‌బుక్

నేను AWS క్లౌడ్‌లో **రోబోషాప్ (Roboshop) ఈ-కామర్స్ సైట్‌ను** విజయవంతంగా క్రియేట్ చేశాను. ఈ మొత్తం ప్రాసెస్‌ను సులువుగా అర్థం చేసుకోవడానికి మరియు ఎగ్జిక్యూట్ చేయడానికి **3 భాగాలుగా (Phases)** విభజించాను.

మొదటి రెండు ఫేజ్‌లను మన లోకల్ సిస్టమ్ నుండి AWS ఇన్‌స్టాన్సెస్ పై రన్ చేశాము, మరియు మూడవ ఫేజ్‌ను సెంట్రల్‌గా ఫ్రంటెండ్ సర్వర్ నుండి ఎగ్జిక్యూట్ చేశాము.

## 📌 ఫేజ్ 1 (Phase 1) — ఇన్‌స్టాన్స్ ప్రొవిజనింగ్ (Installed Instances)

**రన్ చేసే చోటు:** మీ సొంత కంప్యూటర్ (Local Workstation) నుండి AWS పై.

ఈ మొదటి భాగంలో, నేను AWS క్లౌడ్‌లో అప్లికేషన్‌కు కావలసిన 11 లైనక్స్ సర్వర్లను మరియు వాటి నెట్‌వర్క్‌ను క్రియేట్ చేశాను.

* **కమాండ్:** `./provision-infra.sh`
* **ఇది ఏం చేస్తుంది?:** 1. 11 సర్వర్లకు విడివిడిగా సెక్యూరిటీ గ్రూపులను (SG) క్రియేట్ చేస్తుంది.
2. ఒకేసారి 11 EC2 ఇన్‌స్టాన్సెస్ లాంచ్ చేస్తుంది.
3. సర్వర్లు పూర్తిగా ఆన్ అయ్యి, AWS 2/2 Status Checks పాస్ అయ్యేంతవరకు **AWS Waiter** ద్వారా వేచి ఉంటుంది.

## 🌐 ఫేజ్ 2 (Phase 2) — డీఎన్ఎస్ అప్‌డేట్ (Updated IPs in DNS)

**రన్ చేసే చోటు:** మీ సొంత కంప్యూటర్ (Local Workstation) నుండి AWS పై.

సర్వర్లు ఆన్ అయిన తర్వాత వాటికి లభించిన ఐపి అడ్రస్‌లను (IP Addresses) గుర్తుపెట్టుకోవడం కష్టం కాబట్టి, వాటిని డొమైన్ నేమ్స్‌కి మ్యాప్ చేశాను.

* **కమాండ్:** `./update-dns.sh`
* **ఇది ఏం చేస్తుంది?:** 1. కొత్తగా క్రియేట్ అయిన సర్వర్ల ప్రైవేట్ IPలను ఆటోమేటిక్‌గా రీడ్ చేసి, AWS Route 53 లోని మీ డొమైన్‌కు `UPSERT` (A Records) ద్వారా లింక్ చేస్తుంది (ఉదాహరణకు: `catalog.yourdomain.com`).
2. `dispatch` సర్వర్‌కు డొమైన్ అవసరం లేదు కాబట్టి, దాని ప్రైవేట్ ఐపిని మాత్రం నోట్ చేసి ఉంచుకుంటుంది.

### 🔒 సెక్యూర్ కమ్యూనికేషన్ రూల్ (Allowed Frontend Private IP):

ఈ ఫేజ్‌లోనే నేను ఒక ముఖ్యమైన సెక్యూరిటీ రూల్ సెట్ చేశాను. **ఫ్రంటెండ్ సర్వర్ యొక్క Private IP ని మిగతా అన్ని ఇన్‌స్టాన్సెస్ యొక్క SSH సెక్యూరిటీ గ్రూప్‌లోకి ఆటోమేటిక్‌గా అనుమతించాను (Allowed Frontend Private IP to all instances in SG)**. దీనివల్ల ఫ్రంటెండ్ సర్వర్‌కు మిగతా అన్ని సర్వర్లతో కమ్యూనికేట్ అయ్యే పర్మిషన్ లభిస్తుంది.

## 📦 ఫేజ్ 3 (Phase 3) — సాఫ్ట్‌వేర్ ఇన్‌స్టాలేషన్ & సెంట్రల్ ఆర్కెస్ట్రేషన్ (Installed Softwares & Loaded Schema)

**రన్ చేసే చోటు:** ఫ్రంటెండ్ (Frontend) సర్వర్ లోపల నుండి.

నేను ప్రతి సర్వర్‌లోకి విడివిడిగా లాగిన్ అవ్వకుండా, కేవలం **Frontend సర్వర్** లోకి SSH ద్వారా లాగిన్ అయ్యి, అక్కడి నుంచే మిగతా 10 సర్వర్లలో సాఫ్ట్‌వేర్‌లను ఇన్‌స్టాల్ చేశాను. ఫేజ్ 2 లో మనం ఫ్రంటెండ్ ప్రైవేట్ ఐపిని అన్ని సెక్యూరిటీ గ్రూపులలో (SG) అనుమతించడం వల్లే, ఈ మూడవ ఫేజ్‌ను ఫ్రంటెండ్ సర్వర్ నుండి సెంట్రల్‌గా ఎగ్జిక్యూట్ చేయడం సాధ్యపడింది.

* **కమాండ్:** `bash deploy-all.sh`
* **ఇది ఏం చేస్తుంది?:**
1. **Installed Softwares:** బ్యాక్‌గ్రౌండ్‌లో మిగతా అన్ని సర్వర్లకు కనెక్ట్ అయ్యి MongoDB, Redis, MySQL, RabbitMQ లతో పాటు Node.js, Java, Python, Go Lang ఆధారిత అప్లికేషన్ సర్వీసులను ఇన్‌స్టాల్ చేస్తుంది.
2. **Loaded Schema:** అప్లికేషన్‌కు కావలసిన డేటాబేస్ స్కీమాలను (Schema) ఆయా డేటాబేస్‌లలోకి ఆటోమేటిక్‌గా లోడ్ చేస్తుంది.
3. **Frontend Nginx:** చివరగా లోకల్ ఫ్రంటెండ్ సర్వర్ లోపల Nginx వెబ్ సర్వర్‌ను మరియు రివర్స్ ప్రాక్సీని కాన్ఫిగర్ చేస్తుంది.

🔄 ఫెయిల్యూర్ రికవరీ (Resuming from failure) — ఒకవేళ స్క్రిప్ట్ మధ్యలో ఆగితే?
ఈ ఆటోమేషన్‌లో Idempotency (ఒకే పనిని మళ్లీ చేసినా ఎర్రర్ రాకుండా ఉండటం) ఫీచర్ ఉంది. ఒకవేళ ఇన్‌స్టాలేషన్ మధ్యలో (ఉదాహరణకు 8వ సర్వర్ అయిన shipping దగ్గర) నెట్‌వర్క్ లేదా ఇంటర్నెట్ సమస్య వల్ల స్క్రిప్ట్ ఆగిపోతే, మీరు కంగారు పడక్కర్లేదు.

సమస్య ఏంటో చూసి దానిని సరిచేసిన (Fix) తర్వాత, మళ్ళీ bash deploy-all.sh అని రన్ చేస్తే చాలు.

ఇది వరకు విజయవంతంగా పూర్తయిన సర్వర్లను స్క్రిప్ట్ ఆటోమేటిక్‌గా వదిలేస్తుంది (స్కిప్ చేస్తుంది).

నేరుగా ఎక్కడైతే ఇన్‌స్టాలేషన్ ఆగిపోయిందో, అక్కడి నుంచే మళ్ళీ సాఫ్ట్‌వేర్ ఇన్‌స్టాల్ చేయడం స్టార్ట్ చేస్తుంది. దీనికోసం ఇది /var/tmp/roboshop_deploy_state.txt అనే ఫైల్‌లో స్టేటస్‌ను సేవ్ చేసి పెట్టుకుంటుంది.
